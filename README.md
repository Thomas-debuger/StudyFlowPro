## 專案簡介

StudyFlow Pro 是一套為學生設計的個人化學習系統。使用者可以建立自己的帳號，管理課程、任務、課表與考古題，並利用專注計時、智慧分數與智慧排程安排每天的學習順序。

系統所有核心資料皆儲存在本機，不需要登入雲端服務，也不需要網路才能使用。不同帳號會使用不同的資料空間，任務、課表、考古題、專注紀錄、智慧排程、視覺風格與個人設定都不會互相混用。

---

## 系統特色

- **個人帳號系統**：支援註冊、登入、登出與顯示名稱。
- **帳號資料隔離**：每個帳號擁有獨立的任務、課程、課表、專注紀錄、考古題與設定。
- **課程與任務管理**：先建立課程／專案，再將具體任務歸類到對應課程。
- **專注計時**：可綁定任務、暫停、重設，並記錄品質、分心次數與反思。
- **智慧優先分數**：依期限、優先級、剩餘工作量、難度與進度產生 0～100 分。
- **智慧排程**：依可用時間、休息時間與任務條件產生當日學習路線。
- **課表管理**：管理多學期課表，支援 CSV 匯入、CSV 匯出與 PNG 匯出。
- **考古題庫**：管理 PDF／DOCX 考古題、科目分類、閱讀狀態、收藏與筆記。
- **學習分析中心**：視覺化呈現專注時間、品質、課程進度與任務優先級。
- **Research Center**：提供研究型摘要、稽核軌跡、智慧四象限、週報與行事曆匯出。
- **六套視覺風格**：Facebook、Spotify、YouTube、Netflix、Visual Studio、Visual Studio Code。
- **資料安全**：原子式存檔、last-good 保護、滾動快照、匯入／匯出與資料健檢。

## 執行方式

### 使用 Visual Studio 執行

1. 將專案 ZIP 完整解壓縮。
2. 使用 Visual Studio 2022 開啟 `StudyFlowPro.sln`。
3. 等待 Visual Studio 自動還原 NuGet 套件。
4. 選擇上方選單「建置 → 重建方案」。
5. 按 `F5` 或上方綠色開始按鈕執行。
6. 程式會先顯示登入頁面；第一次使用請按「建立新帳號」。

### 使用已編譯版本執行

若資料夾內已有完整發佈版本，直接雙擊：

```text
StudyFlowPro.exe
```

請勿只複製單一 `.exe`，應保留同一資料夾內的 DLL、設定檔與相依檔案。

## 重點功能詳細介紹

以下 12 項功能依照主畫面左側工具列由上到下排列。每個功能皆使用目前登入帳號的個人資料；除「寄信詢問教授」共用元智大學資工系教授名錄外，其餘課程、任務、課表、考古題、專注紀錄、智慧排程、分析結果、視覺風格與設定都會依帳號分開保存。

### 1. 主控台

<img width="1918" height="1140" alt="StudyFlow Pro 主控台" src="https://github.com/user-attachments/assets/1bbce95a-fe7d-4c79-9f82-f3c16c32dfff" />

主控台是登入後的首頁，用來快速掌握今天最重要的學習資訊。

- **摘要卡片**：顯示未完成任務、逾期任務、任務完成率與今日專注分鐘。
- **SMART PRIORITY 智慧建議**：從未完成任務中選出目前最值得優先處理的項目，並顯示課程、剩餘期限、建議優先級與智慧分數。
- **今日作戰計畫**：顯示智慧排程最近一次產生並儲存的結果，包括可用時間、專注時間、休息時間、預計結束時間與任務順序。
- **下一步任務**：列出近期應處理的任務、截止時間、進度與智慧分數，方便直接決定下一步。
- **自動更新**：任務完成、專注紀錄或智慧排程變更後，主控台會重新載入目前帳號的最新資料。

### 2. 課程／專案

<img width="1918" height="1137" alt="StudyFlow Pro 課程與專案" src="https://github.com/user-attachments/assets/2ae36fd1-3884-4823-8b53-90cb82101284" />

課程／專案是任務的分類中心，用來建立長期學習主題，例如「演算法」、「作業系統」或「專題研究」。

- 可新增、編輯或刪除課程／專案。
- 可設定**名稱、老師／負責人、地點與識別顏色**。
- 課程列會直接使用所選的原始色碼顯示，深色背景自動使用白字，淺色背景自動使用黑字。
- 點選某一門課程後，下方會立即列出歸類在該課程底下的任務。
- 下方任務表會顯示狀態、優先級、截止時間、進度與智慧分數；雙擊任務可直接編輯。
- 課程完成率會依該課程底下所有任務的實際進度平均計算，已完成任務以 100% 計算。

### 3. 任務管理

<img width="1918" height="1140" alt="StudyFlow Pro 任務管理" src="https://github.com/user-attachments/assets/83ef26a8-57d0-41e5-8ff9-7a3b827d9272" />

任務管理用來記錄真正需要完成的工作，例如「複習 DFS」、「完成期末專案」或「整理研究圖表」。

- 可新增、編輯、完成／取消完成、釘選／取消釘選與刪除任務。
- 每個任務可設定**所屬課程、優先級、截止時間、預估分鐘、難度、精力需求、標籤、說明與目前進度**。
- 課程欄位會使用該課程的識別顏色，方便快速辨識。
- 可依名稱、標籤或說明搜尋，也可依狀態篩選。
- **優先原因**會說明智慧分數由哪些條件組成，例如期限、剩餘工作量、難度、釘選與進度。
- **智慧排程**可直接使用目前任務產生當日學習路線。
- 支援**匯入任務 CSV**與**匯出任務 CSV**，方便批次建立、備份或轉移任務。

### 4. 專注計時

<img width="1918" height="1141" alt="StudyFlow Pro 專注計時" src="https://github.com/user-attachments/assets/59c3b145-6ada-4849-823f-dd6299630f15" />

專注計時將實際學習時間寫入紀錄，也可同步回饋任務進度。

- 可選擇綁定既有任務，也可使用「自由專注」。
- 可直接選擇 25、50、90 分鐘，或自行輸入分鐘數。
- 提供**開始／繼續、暫停、完成並記錄、重設**等操作。
- 重設前會跳出確認視窗；按右上角 X、取消或 Esc 都不會清除目前進度。
- 完成專注後可記錄**品質 1～5 分、分心次數與備註／反思**。
- 綁定任務時，專注分鐘會回寫到該任務，並更新其實際進度。
- 下方顯示最近專注紀錄，包括日期、任務、課程、分鐘、品質、分心與備註。
- 支援**匯入紀錄 CSV**與**匯出紀錄 CSV**。

### 5. 課表

<img width="1918" height="1138" alt="StudyFlow Pro 課表" src="https://github.com/user-attachments/assets/7b18ba78-b16d-44ad-8143-21fe841e17ba" />

課表功能用來管理不同學期的固定上課時段。

- 可建立、切換與刪除不同學期，例如「114-2」或「115-1」。
- 可新增、編輯或刪除課程，並設定星期、節次、課程名稱、教室與顏色。
- 雙擊課表中的課程格可直接編輯；雙擊空白格可新增課程。
- 同一時段若出現多門課，系統會進行衝突檢查。
- 上方統計會顯示目前學期、不重複課程數、每週上課節數與今天課程。
- 支援**匯入課表 CSV、匯出課表 CSV、匯出課表 PNG**。
- 「新增學期課表」可建立空白學期，再匯入既有 CSV 或自行新增課程。

### 6. 考古題庫

<img width="1918" height="1142" alt="StudyFlow Pro 考古題庫" src="https://github.com/user-attachments/assets/a725e9fa-60ba-4999-960d-7e2f0a03f7b3" />

考古題庫用來分類、保存與追蹤 PDF／DOCX 考古題的複習狀態。

- 可建立、編輯或刪除科目，並替每個科目設定識別顏色。
- 可匯入 PDF／DOCX，也可將檔案拖曳到題庫列表快速加入。
- 提供搜尋、格式篩選、閱讀狀態篩選與收藏功能。
- 可切換「未開始、複習中、已完成」等狀態，並記錄開啟次數。
- 可預覽檔案、編輯標題、年份、類型、標籤與筆記，或開啟／匯出原始檔。
- **智慧抽一份**可從目前題庫中挑選一份考古題開始複習。
- **檔案完整性警示**可檢查索引存在但原始檔遺失等問題。
- 支援題庫包匯出與匯入，可連同索引及 PDF／DOCX 原始檔一起轉移。
- 每個帳號都有自己的考古題科目、閱讀狀態、筆記與原始檔。

### 7. 寄信詢問教授

<img width="1918" height="1140" alt="StudyFlow Pro 寄信詢問教授" src="https://github.com/user-attachments/assets/64e80e68-2bcf-4c6d-9342-f25c5f80d55a" />

寄信詢問教授整合元智大學資工系教授名錄與正式信件範本，協助學生快速完成郵件草稿。

- 可依教授姓名、信箱或研究領域搜尋，並依領域篩選。
- 左側顯示教授姓名、職稱、電子郵件與辦公室；右側顯示更完整的教授資料。
- 若課程已設定老師，系統可在右上角提示目前課程教師。
- 可選擇不同信件範本，再修改主旨與內容。
- **複製信箱**可快速複製教授電子郵件。
- **使用 Gmail 撰寫**會開啟 Gmail 撰寫頁面並帶入收件者、主旨與內容。
- 系統只負責產生郵件草稿，不會保存 Gmail 帳號或密碼。
- 教授名錄為全帳號共用；寄件者顯示名稱則依目前登入帳號帶入。

### 9. 智慧排程

<img width="1918" height="1138" alt="StudyFlow Pro 智慧排程" src="https://github.com/user-attachments/assets/bc4cd988-adb4-46bd-a467-b04a46587b50" />

智慧排程依任務智慧分數、截止時間與剩餘工作量，自動產生可執行的今日學習路線。

- 可設定**開始時間、可用分鐘與每次休息分鐘**。
- 「使用現在時間」可快速把起始時間改成目前時間。
- 「重新產生」會依最新條件重新安排，並立刻保存至目前帳號。
- 上方摘要顯示專注區塊數、專注總分鐘、保留緩衝與預計結束時間。
- 每個排程區塊會顯示時間、任務、課程、智慧分數、優先級、區塊目標與休息時間。
- 右側列出目前首要任務與智慧分數圖例。
- **可解釋規則式排程**會說明系統採用的排序原則。
- 支援**匯出 TXT**與**複製今日計畫**。
- 關閉視窗後，最新排程仍會顯示在主控台的「今日作戰計畫」。
- 不同帳號的可用分鐘、休息分鐘與排程結果彼此獨立。

### 10. 學習分析中心

<img width="1918" height="1137" alt="StudyFlow Pro 學習分析中心" src="https://github.com/user-attachments/assets/2d2aef14-f2bb-45f0-bdd4-6cf55e0e6f38" />

學習分析中心用來回答：「我最近學得怎麼樣？」

#### 上方四個指標

| 指標 | 說明 |
|---|---|
| **生產力指數** | 0～100 分的綜合分數，整合任務完成率、學習一致性、專注目標達成率與專注品質。 |
| **本週專注** | 統計本週星期一到今天，所有有效專注紀錄的分鐘總和。 |
| **專注品質** | 將本週每次專注後的 1～5 分自評取平均，再換算成百分比。 |
| **估時準確度** | 比較已完成任務的預估分鐘與實際專注分鐘；越接近 100% 代表估時越準，資料不足時顯示「待收集」。 |

生產力指數公式：

```text
生產力指數
＝任務完成率 × 30%
＋學習一致性 × 25%
＋專注目標達成率 × 25%
＋專注品質 × 20%
```

估時準確度的單一任務計算方式：

```text
單一任務準確度＝較小時間 ÷ 較大時間 × 100%
```

其他圖表：

- **近 7 天專注分鐘與品質**：長條高度代表每日專注分鐘；`Q4.0` 表示當日平均品質為 4.0／5。
- **各課程完成率**：依該課程底下所有任務的實際進度平均值計算，已完成任務以 100% 計算。
- **未完成任務優先級分布**：統計立即處理、高度優先、中度優先與可排程任務的數量及比例。

### 11. Research Center

<img width="1918" height="1138" alt="StudyFlow Pro Research Center" src="https://github.com/user-attachments/assets/093fdcc1-54b4-41e7-9899-9d91c31c42f8" />

Research Center 用來回答：「我的研究或長期學習流程是否穩定，資料是否完整？」

#### 上方四個指標

| 指標 | 說明 |
|---|---|
| **生產力指數** | 與學習分析中心相同，呈現任務完成、規律程度、目標達成與專注品質的綜合表現。 |
| **學習一致性** | 本週已經過的天數中，有多少天至少留下過一筆有效專注紀錄。 |
| **專注品質** | 本週所有品質評分的平均百分比。 |
| **估時準確度** | 比較已完成任務的預估與實際專注時間；資料不足時顯示「待收集」。 |

學習一致性公式：

```text
學習一致性
＝有專注紀錄的天數 ÷ 本週已經過天數 × 100%
```

- **研究型摘要**：依生產力指數、本週與上週專注變化、學習一致性和專注品質，產生可解釋的建議文字。它不是生成式 AI，而是規則式統計摘要。
- **深度工作分鐘**：單次專注至少 25 分鐘，且分心次數不超過 1 次。
- **稽核軌跡**：記錄新增、修改、閱讀、匯入、匯出與切換風格等近期系統操作，方便追蹤資料變動。

底部按鈕：

| 按鈕 | 功能 |
|---|---|
| **輸出 HTML 週報** | 將指標、專注趨勢、課程進度、摘要與優先任務整理成離線網頁，可列印或另存為 PDF。 |
| **智慧四象限** | 依重要性與緊急程度，自動將未完成任務分為四類。 |
| **資料健檢** | 檢查帳號所有權、重複 ID、任務關聯、課表衝突、考古題檔案與專注紀錄等問題，不會擅自修改資料。 |
| **匯出 iCalendar** | 將未完成任務與截止時間輸出為 `.ics`，可匯入 Google Calendar、Outlook 或 Apple Calendar。 |
| **關閉** | 關閉 Research Center 並返回主系統，不會登出帳號。 |

### 12. 設定與備份

<img width="1918" height="1137" alt="StudyFlow Pro 設定與資料管理" src="https://github.com/user-attachments/assets/ebdf7b20-bcc2-47a7-abc2-256dd52d6bb4" />

設定與備份視窗用來管理目前帳號的個人偏好與本機資料安全。

#### 個人設定

- **顯示名稱**：同步顯示於主畫面問候語、左上角帳號區與視窗標題。
- **預設專注分鐘**：設定專注計時的預設時長。
- **每日專注目標**：用於計算專注目標達成率與生產力指數。
- **啟動程式時顯示目前最緊急任務**：登入後主動提醒。
- **即將到期判定範圍**：設定未來幾小時內到期的任務要視為即將到期。

#### 資料安全中心按鈕

## 第一次使用

1. 在登入頁面按「還沒有帳號？建立新帳號」。
2. 輸入顯示名稱、登入帳號、密碼與確認密碼。
3. 按「建立帳號並登入」。
4. 先到「課程／專案」建立課程，例如演算法、作業系統或專題研究。
5. 到「任務管理」新增具體工作，並設定課程、期限、優先級、預估時間與進度。
6. 可到「課表」建立個人學期課表。
7. 使用「專注計時」執行任務並留下學習紀錄。
8. 回到「主控台」、「分析中心」或「Research Center」查看成果。

## 資料儲存與帳號隔離

系統資料預設儲存在：

```text
%LocalAppData%\StudyFlowPro\
```

主要結構：

```text
%LocalAppData%\StudyFlowPro\
├─ Auth\
│  └─ accounts.json
└─ Users\
   └─ <帳號ID>\
      ├─ studyflow-data.json
      ├─ studyflow-data.lastgood.json
      ├─ profile-settings.json
      ├─ profile-settings.lastgood.json
      ├─ Snapshots\
      └─ ExamLibrary\
         ├─ Files\
         └─ PreviewCache\
```

- `accounts.json`：保存帳號資料與密碼雜湊，不保存明碼密碼。
- `studyflow-data.json`：保存任務、課程、課表、專注紀錄、智慧排程、考古題索引與活動紀錄。
- `profile-settings.json`：保存目前帳號的視覺風格、專注設定、提醒與個人偏好。
- `Snapshots`：保存自動滾動快照。
- `ExamLibrary\Files`：保存匯入的 PDF／DOCX 原始檔。

### 帳號隔離原則

不同帳號的下列資料均互相獨立：

- 課程／專案與任務
- 專注紀錄
- 課表與學期
- 考古題科目、閱讀狀態與原始檔
- 智慧排程
- 視覺風格
- 學習分析與 Research Center 統計
- 個人設定、匯入資料與快照

「寄信詢問教授」的元智資工教授名錄為全帳號共用，因為所有使用者皆使用相同學校教授資料；寄件者顯示名稱仍會依目前登入帳號帶入。

## 使用注意事項

- 本系統為 Windows 桌面應用程式，不支援 macOS 或 Linux 直接執行。
- 第一次開啟方案時需等待 NuGet 還原完成。
- PDF 內嵌預覽需要 Microsoft Edge WebView2 Runtime。
- 系統資料位於使用者的 `%LocalAppData%`，不會因壓縮原始碼專案而自動包含在 ZIP 中。
- 交作業前建議先建立新帳號，測試登入、任務、課表、專注計時、智慧排程、分析中心與資料匯出功能。
- 請勿將 `.vs`、`bin`、`obj` 或 `.git` 資料夾放入繳交 ZIP。
